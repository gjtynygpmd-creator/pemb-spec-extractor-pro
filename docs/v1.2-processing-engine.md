# v1.2 Processing Engine MVP

## Purpose
Move analysis jobs beyond `queued` by adding a persistent Render background worker.

## Workflow
1. API creates a queued job.
2. Worker atomically claims the oldest queued job.
3. Worker downloads PDFs from Cloudflare R2.
4. PyMuPDF reads searchable text and page metadata.
5. Pages are classified and relevant CSI divisions are detected.
6. Initial PEMB fields are extracted with source page references.
7. Conflicting values are flagged for review.
8. Neon stores pages, events, fields, and status updates.
9. Netlify polls the workspace while a job is active.

## Supported in this MVP
- Searchable PDFs
- Division 05 and Division 13 as primary targets
- Division 07 insulation and Division 08 doors/openings as important optional targets
- Structural/general notes, roof plans, framing plans, elevations, wall sections, and door schedules

## Deferred
- OCR for image-only pages
- Drawing vision and dimension recognition
- AI semantic extraction
- User authentication and subscriptions
